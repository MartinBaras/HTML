Introducción

¿Qué son, para qué sirven y por qué aprenderlos?

🧩 ¿Qué son los Web Components?

Los Web Components son un conjunto de APIs nativas del navegador que permiten crear elementos HTML personalizados, encapsulados y completamente reutilizables, usando solo JavaScript vanilla, sin necesitar ningún framework externo (ni React, ni Vue, ni Angular).

La idea central es simple pero poderosa: así como el navegador tiene sus propias etiquetas como <input>, <video> o <details>, con los Web Components vos podés crear las tuyas propias: <mi-boton>, <tarjeta-usuario>, <modal-confirmacion>.

💡
Analogía didáctica: Pensá en los Web Components como "piezas de LEGO personalizadas". Cada pieza tiene su propia lógica interna, se puede usar en cualquier proyecto y no interfiere con las demás piezas. Una vez definida, la usás como cualquier etiqueta HTML.

Son un estándar del W3C, lo que significa que todos los navegadores modernos los soportan de forma nativa, sin polyfills ni dependencias adicionales.

🏛️ Los 3 pilares fundamentales

Los Web Components se construyen sobre tres tecnologías del navegador que trabajan juntas:

1. Custom Elements

Define nuevas etiquetas HTML con comportamiento propio. Extendés la clase HTMLElement y registrás tu elemento con un nombre personalizado.

2. Shadow DOM

Encapsula el HTML y CSS interno del componente, creando un árbol DOM aislado que no es afectado por los estilos globales de la página.

3. HTML Templates

Define fragmentos HTML reutilizables que no se renderizan hasta que se necesitan. Son el "molde" o "esqueleto" del componente.

4. ES Modules (bonus)

Importa y exporta componentes entre archivos usando el sistema de módulos nativo del navegador. Permite organizar el código en archivos separados.

⚖️ ¿Por qué aprenderlos? Ventajas y limitaciones

✓ Por qué sí usarlos

  • Nativo del navegador, sin instalar nada
  • Sin dependencias externas que actualizar
  • Encapsulación CSS real con Shadow DOM
  • Reutilizable en cualquier framework
  • Estándar web oficializado (W3C)
  • Largo soporte garantizado en el futuro
  • Tamaño de bundle: cero kilobytes extra

✗ Limitaciones a conocer

  • Más verboso que React/Vue/Svelte
  • No tiene Virtual DOM ni reactividad
  • Ecosistema de herramientas más pequeño
  • Sin hot module reload nativo
  • Server-side rendering es más complejo
  • La curva inicial es más empinada
ℹ️
¿Reemplazan a React o Vue? No necesariamente. Los Web Components son excelentes para crear librerías de componentes agnósticas al framework, o para proyectos sin build process. React/Vue son mejores para aplicaciones complejas con estado global y ecosistema rico.

Custom Elements

Cómo definir y registrar tus propias etiquetas HTML

🏷️ Definiendo un Custom Element

Un Custom Element se crea en dos pasos: (1) definir una clase JavaScript que extiende HTMLElement, y (2) registrarla con customElements.define().

⚠️
Regla obligatoria: El nombre siempre debe contener al menos un guion (ej: mi-boton, app-header). Esto es una especificación del estándar para evitar colisiones con etiquetas HTML nativas actuales o futuras. miboton es inválido.

Creamos una clase que hereda de HTMLElement. El constructor se llama cuando el elemento es creado.

// Toda clase de Web Component debe extender HTMLElement
class MiBoton extends HTMLElement {

  constructor() {
    super(); // ← SIEMPRE llamar a super() como primera línea
    
    // Aquí configuramos el contenido inicial del componente
    this.innerHTML = '<button>¡Soy un botón personalizado!</button>';
  }
}
💡
¿Por qué super()? Al extender una clase, super() llama al constructor padre (HTMLElement). Sin esta llamada, this no estaría disponible y obtendríamos un error. Es como decirle al navegador "primero inicializame como un elemento HTML normal, y luego agrego mi lógica extra".

Una vez definida la clase, la registramos para que el navegador sepa qué clase usar cuando encuentre esa etiqueta.

// customElements.define( nombre-etiqueta, clase )
customElements.define(
  'mi-boton',  // nombre de la etiqueta (con guion, en minúsculas)
  MiBoton      // la clase que define su comportamiento
);

// A partir de este momento, el navegador reconoce <mi-boton>
// También podemos verificar si ya fue registrado:
console.log(customElements.get('mi-boton')); // → class MiBoton
⚠️
Un nombre, una vez: Intentar registrar el mismo nombre dos veces lanza un error. Si cargás el archivo dos veces (por ejemplo en desarrollo), usá if (!customElements.get('mi-boton')) antes de registrar.

Una vez registrado, lo usamos como cualquier etiqueta HTML en nuestros archivos:

<!-- En el HTML, usarlo es idéntico a cualquier etiqueta nativa -->
<mi-boton></mi-boton>
<mi-boton></mi-boton>
<mi-boton></mi-boton>

<!-- O crearlo dinámicamente desde JavaScript -->
const boton = document.createElement('mi-boton');
document.body.appendChild(boton);

<!-- El script del componente debe cargarse antes (o en el mismo archivo) -->
<script src="mi-boton.js" type="module"></script>
🎛️ Trabajando con atributos

Los componentes pueden recibir datos desde el HTML a través de atributos, exactamente igual que un <input type="text" placeholder="..."> usa atributos para configurarse.

class TarjetaUsuario extends HTMLElement {

  connectedCallback() {
    // getAttribute() lee el valor de un atributo HTML
    const nombre = this.getAttribute('nombre') || 'Usuario';
    const rol    = this.getAttribute('rol')    || 'Sin rol';
    const avatar = this.getAttribute('avatar') || '👤';

    this.innerHTML = `
      <div style="display:flex; gap:.75rem; align-items:center;
                  padding:1rem; border:1px solid #ddd; border-radius:10px;">
        <span style="font-size:2rem;">${avatar}</span>
        <div>
          <strong>${nombre}</strong>
          <p style="color:#888; font-size:.85em; margin:0;">${rol}</p>
        </div>
      </div>
    `;
  }
}

customElements.define('tarjeta-usuario', TarjetaUsuario);

<!-- Los atributos se pasan directamente en el HTML -->
<tarjeta-usuario nombre="Ana García" rol="Docente" avatar="👩‍🏫"></tarjeta-usuario>
<tarjeta-usuario nombre="Carlos López" rol="Estudiante" avatar="👨‍💻"></tarjeta-usuario>
💡
Atributos vs Propiedades: Los atributos son strings en el HTML (getAttribute()). Las propiedades son valores JavaScript (element.nombre). Para datos complejos (objetos, arrays) conviene usar propiedades JS; para datos simples (texto, números, booleans) usan atributos HTML.

Shadow DOM

Encapsulación real: CSS que no se "escapa" ni "se contamina"

🔒 El problema que resuelve el Shadow DOM

Uno de los mayores dolores de cabeza en CSS es el alcance global. Si definís p { color: red; } en tu hoja de estilos, todos los párrafos de la página se vuelven rojos, incluyendo los que están dentro de tus componentes.

El Shadow DOM resuelve esto creando un árbol DOM separado y aislado dentro del elemento. El CSS del exterior no entra; el CSS del interior no sale.

DOM Principal (Light DOM)
<body>
<h1>Mi página</h1>
<mi-widget>
⬆ Tu acceso termina aquí
</mi-widget>
</body>
→
Shadow Tree (aislado)
<style> ← solo aplica aquí
<button>Interno</button>
<span class="dato">42</span>
No afectado por CSS externo
ℹ️
Ya lo usás sin saberlo: Las etiquetas <video>, <input type="range">, <details> y <input type="date"> usan Shadow DOM internamente para renderizar sus controles. En DevTools podés activar "Show user agent shadow DOM" para verlo.
💻 Cómo usar attachShadow()

Para activar el Shadow DOM en un componente, llamamos a this.attachShadow() en el constructor. Esto retorna el Shadow Root, que es el nodo raíz del árbol aislado.

class PanelAislado extends HTMLElement {
  constructor() {
    super();

    // attachShadow crea y retorna el Shadow Root
    // mode 'open':   element.shadowRoot → devuelve el Shadow Root (accesible)
    // mode 'closed': element.shadowRoot → devuelve null   (completamente privado)
    const shadow = this.attachShadow({ mode: 'open' });

    // Todo lo que agreguemos al shadow root vive en el Shadow DOM
    shadow.innerHTML = `
      <style>
        /* Este CSS SOLO aplica dentro de este componente.
           Si la página tiene p { color: red }, este p NO se ve afectado. */
        p {
          color: teal;
          font-weight: bold;
          padding: 1rem;
          background: #f0fff4;
          border-radius: 8px;
        }
        :host {
          /* :host selecciona el elemento <panel-aislado> en sí mismo */
          display: block;
          border: 2px solid teal;
          border-radius: 10px;
        }
      </style>

      <p>
        Soy un párrafo dentro del Shadow DOM.
        El CSS externo no puede cambiar mi apariencia.
      </p>
    `;
  }
}

customElements.define('panel-aislado', PanelAislado);
💡
El selector :host: Dentro del Shadow DOM, :host selecciona el propio elemento custom (el "dueño" del shadow). Es el equivalente a seleccionar panel-aislado desde el exterior, pero desde adentro del componente. Sirve para definir el display, bordes, márgenes del elemento contenedor.
🎨 CSS personalizable con propiedades CSS

Aunque el Shadow DOM aísla los estilos, podemos crear "puntos de personalización" usando Custom Properties (variables CSS). Las variables CSS atraviesan la barrera del Shadow DOM.

/* Dentro del componente: usamos la variable con un fallback */
:host {
  background: var(--mi-boton-bg, #1D9E75);  /* fallback: verde */
  color:       var(--mi-boton-color, white);
}

/* Desde el exterior, el usuario personaliza el componente */
mi-boton {
  --mi-boton-bg:    #185FA5;  /* override: azul */
  --mi-boton-color: white;
}

HTML Templates & Slots

Moldes reutilizables y puntos de inserción de contenido

📋 ¿Qué es el elemento <template>?

La etiqueta <template> define HTML que no se renderiza cuando la página carga. El navegador lo parsea y lo tiene disponible en memoria, pero no lo muestra ni ejecuta ningún script o carga ninguna imagen que contenga.

Se "activa" solo cuando lo clonamos explícitamente con JavaScript usando cloneNode(true).

ℹ️
Ventaja de rendimiento: Clonar un nodo ya parseado es mucho más rápido que parsear HTML desde un string (innerHTML) repetidamente. Cuando un componente se usa muchas veces, los templates marcan diferencia.
<!-- 1. Definimos el template en el HTML (se puede poner en el <head> o <body>) -->
<template id="tpl-tarjeta">
  <style>
    .tarjeta {
      border: 1px solid #ddd;
      border-radius: 10px;
      padding: 1.25rem;
      font-family: sans-serif;
    }
    .tarjeta h3 { color: #1D9E75; margin-bottom: .5rem; }
    .tarjeta p  { color: #666; font-size: .9em; }
  </style>
  <div class="tarjeta">
    <h3>Título de la tarjeta</h3>
    <p>Descripción de la tarjeta.</p>
  </div>
</template>

// 2. En el componente, clonamos el template
class TarjetaInfo extends HTMLElement {
  connectedCallback() {
    const shadow    = this.attachShadow({ mode: 'open' });
    const template  = document.getElementById('tpl-tarjeta');

    // cloneNode(true) → copia profunda: incluye todos los elementos hijos
    // cloneNode(false) → copia superficial: solo el nodo raíz, sin hijos
    shadow.appendChild(template.content.cloneNode(true));
  }
}
🔌 Slots: puntos de inserción de contenido

Los <slot> son "agujeros" en el template donde quien usa el componente puede insertar su propio HTML. Son análogos al concepto de "children" en React o "slots" en Vue, y tienen el mismo nombre por razón.

Hay dos tipos:

  • Slot por defecto (<slot>): recibe todo el contenido que no tenga un slot="nombre"
  • Slot nombrado (<slot name="titulo">): recibe solo el contenido con slot="titulo"
<!-- Template con slots nombrados -->
<template id="tpl-articulo">
  <style>
    article { border: 1px solid #eee; border-radius: 10px; overflow: hidden; }
    header  { background: #1D9E75; color: white; padding: 1rem; }
    .cuerpo { padding: 1rem; }
    footer  { background: #f5f5f5; padding: .75rem 1rem; font-size: .85em; color: #888; }
  </style>
  <article>
    <header>
      <slot name="titulo">Sin título</slot>  <!-- slot nombrado -->
    </header>
    <div class="cuerpo">
      <slot></slot>                     <!-- slot por defecto -->
    </div>
    <footer>
      <slot name="pie">Sin fecha</slot>    <!-- slot nombrado -->
    </footer>
  </article>
</template>

<!-- Uso: el atributo slot="nombre" conecta el contenido con su slot -->
<articulo-card>
  <span slot="titulo">Introducción a Web Components</span>
  <p>Este es el contenido del artículo que va en el slot por defecto.</p>
  <span slot="pie">Publicado: 11 de Mayo, 2026</span>
</articulo-card>
💡
El contenido de los slots vive en el Light DOM: Lo que el usuario pone en un slot sigue siendo parte del DOM normal (accesible con querySelector), no del Shadow DOM. El Shadow DOM solo decide dónde mostrarlo visualmente.

Ciclo de vida

Los 4 callbacks que el navegador llama automáticamente

🔄 Lifecycle Callbacks

Los Custom Elements tienen métodos especiales que el navegador invoca automáticamente en distintos momentos de la "vida" del elemento. Son similares a los lifecycle hooks de React (componentDidMount, componentWillUnmount) o Vue (mounted, unmounted), pero son el estándar nativo.

constructor() Creación

Se llama cuando el elemento es creado (ya sea por el parser HTML o por document.createElement()). Siempre debe llamar a super() primero. Usalo para inicializar el Shadow DOM y el estado interno. No leas atributos aquí, porque el elemento aún no está en el DOM y pueden no estar disponibles.

connectedCallback() Insertado en DOM

Se llama cada vez que el elemento es insertado en el documento. Aquí ya tenés acceso a los atributos. Ideal para: leer atributos y renderizar, hacer fetch de datos, suscribirse a eventos externos, iniciar timers. Puede llamarse múltiples veces si el elemento se mueve entre nodos.

attributeChangedCallback(nombre, anterior, nuevo) Atributo cambia

Se llama cuando un atributo observado cambia su valor. Solo se dispara para los atributos declarados en el getter estático observedAttributes. Los tres parámetros son: el nombre del atributo, el valor anterior y el nuevo valor.

disconnectedCallback() Removido del DOM

Se llama cuando el elemento es removido del documento. Usalo para limpiar: cancelar subscripciones a eventos, limpiar timers (clearInterval), cancelar peticiones en curso. Importante para evitar memory leaks.

💻 Todos los callbacks en acción
class ContadorReactivo extends HTMLElement {

  // ① Declaramos qué atributos queremos que disparen attributeChangedCallback
  // Sin esto, los cambios de atributos son ignorados
  static get observedAttributes() {
    return ['valor', 'color'];
  }

  // ② constructor: inicialización, Shadow DOM
  constructor() {
    super();
    this._shadow = this.attachShadow({ mode: 'open' });
    this._shadow.innerHTML = '<span style="font-size:2rem;">0</span>';
    console.log('🔨 constructor: elemento creado');
  }

  // ③ connectedCallback: elemento en el DOM, ya podemos leer atributos
  connectedCallback() {
    console.log('✅ connectedCallback: insertado en el DOM');
    this._render(); // Renderizamos con los atributos actuales
  }

  // ④ attributeChangedCallback: un atributo observado cambió
  attributeChangedCallback(nombre, anterior, nuevo) {
    console.log(`🔄 attributeChangedCallback: "${nombre}" cambió de "${anterior}" a "${nuevo}"`);
    this._render(); // Volvemos a renderizar con el nuevo valor
  }

  // ⑤ disconnectedCallback: limpiamos recursos
  disconnectedCallback() {
    console.log('❌ disconnectedCallback: removido del DOM');
    // Aquí limpiaríamos timers, event listeners, etc.
  }

  // Método privado de renderizado (convención: prefijo _)
  _render() {
    const valor = this.getAttribute('valor') || '0';
    const color = this.getAttribute('color') || 'teal';
    const span  = this._shadow.querySelector('span');
    span.textContent          = valor;
    span.style.color          = color;
  }
}

customElements.define('contador-reactivo', ContadorReactivo);

<!-- Uso: -->
<contador-reactivo valor="42" color="#1D9E75"></contador-reactivo>

// Cambiar el atributo dispara attributeChangedCallback automáticamente
document.querySelector('contador-reactivo').setAttribute('valor', '100');
⚠️
Orden de ejecución importante: Cuando el HTML se parsea, el orden es: constructor → attributeChangedCallback (por cada atributo en el HTML) → connectedCallback. Por eso, si leés atributos en connectedCallback, ya tendrán sus valores finales.

Ejemplo completo

Construimos un componente de alerta desde cero, paso a paso

🔨 Objetivo: <alerta-mensaje>

Vamos a construir un componente que muestre mensajes de alerta con diferentes tipos (éxito, error, aviso, información). Combinará todo lo aprendido: Custom Elements, Shadow DOM, Templates y Slots.

Vista previa del resultado final
✅ ¡Registro completado con éxito!
❌ Error: el servidor no responde.
ℹ️ Hay una nueva versión disponible.
⚠️ Tu sesión expirará en 5 minutos.
📝 Implementación completa
1

Crear el template con slots y estilos

Definimos el esqueleto visual con los slots y el CSS de cada tipo de alerta.

2

Adjuntar el Shadow DOM y clonar el template

En el constructor, creamos el árbol aislado y lo populamos con el template.

3

Observar el atributo "tipo" para reaccionar a cambios

Usamos attributeChangedCallback para actualizar la vista si el tipo cambia.

4

Registrar y usar en HTML

Registramos el componente y lo usamos con slots nombrados.

// ═══════════════════════════════════════════════
// 📁 alerta-mensaje.js — Componente completo
// ═══════════════════════════════════════════════

// ① Creamos el template programáticamente
const template = document.createElement('template');
template.innerHTML = `
  <style>
    :host {
      display: block;
      margin: .5rem 0;
      font-family: sans-serif;
    }

    .alerta {
      padding: .75rem 1rem;
      border-radius: 8px;
      font-size: 14px;
      display: flex;
      align-items: flex-start;
      gap: .6rem;
      line-height: 1.5;
    }

    /* Estilos por tipo usando :host([attr]) */
    :host([tipo="exito"])   .alerta { background:#d4edda; color:#155724; border:1px solid #c3e6cb; }
    :host([tipo="error"])   .alerta { background:#f8d7da; color:#721c24; border:1px solid #f5c6cb; }
    :host([tipo="info"])    .alerta { background:#d1ecf1; color:#0c5460; border:1px solid #bee5eb; }
    :host([tipo="aviso"])   .alerta { background:#fff3cd; color:#856404; border:1px solid #ffeeba; }

    /* Tipo por defecto si no se especifica */
    .alerta { background:#f8f9fa; color:#383d41; border:1px solid #d6d8db; }

    .icono { font-size: 1rem; flex-shrink: 0; margin-top: 1px; }
    .contenido { flex: 1; }
  </style>

  <div class="alerta">
    <span class="icono">
      <slot name="icono">💬</slot>
    </span>
    <div class="contenido">
      <slot>Mensaje de alerta por defecto.</slot>
    </div>
  </div>
`;

// ② Definimos la clase del componente
class AlertaMensaje extends HTMLElement {

  static get observedAttributes() {
    return ['tipo']; // ③ Observamos el atributo "tipo"
  }

  constructor() {
    super();
    // Adjuntamos el Shadow DOM y clonamos el template
    const shadow = this.attachShadow({ mode: 'open' });
    shadow.appendChild(template.content.cloneNode(true));
  }

  // Si el atributo "tipo" cambia, el CSS ya reacciona por sí solo
  // gracias al selector :host([tipo="..."]) — no necesitamos JS extra
  attributeChangedCallback(nombre, anterior, nuevo) {
    console.log(`Tipo cambió: ${anterior} → ${nuevo}`);
  }
}

// ④ Registramos el componente
customElements.define('alerta-mensaje', AlertaMensaje);

Antes de empezar:
qué necesitás saber

Para trabajar con APIs necesitamos entender tres conceptos clave: cómo funciona HTTP, qué es JSON y cómo maneja JavaScript las operaciones que tardan tiempo.

🌐
HTTP
El protocolo que usa el navegador para pedir y recibir datos de servidores remotos. GET, POST, PUT, DELETE.
📦
JSON
El formato de datos que usan las APIs. JavaScript Object Notation — ya lo conocés, es casi idéntico a un objeto JS.
⏳
Asincronía
Fetch tarda tiempo. JS no puede bloquearse esperando. Por eso usamos Promises y async/await.
🔗
API REST
Una URL que devuelve datos en JSON. Rick and Morty API: rickandmortyapi.com

HTTP en 30 segundos

Cuando tu navegador pide una página, hace una petición HTTP. Las APIs funcionan igual: mandás una petición a una URL, y el servidor responde con datos en vez de HTML.

Una petición HTTP básica en la consola del navegador
Concepto
// Esto es lo que hace fetch() internamente:
// → GET https://rickandmortyapi.com/api/character/1
// ← 200 OK
// ← Content-Type: application/json
// ← { "id": 1, "name": "Rick Sanchez", ... }

// Códigos HTTP importantes:
// 200 = OK        ✓ todo bien
// 404 = Not Found ✗ la URL no existe
// 500 = Server Error ✗ algo rompió el servidor

Async / Promise — la clave de todo

Fetch devuelve una Promise: una promesa de que "en algún momento voy a tener un resultado". Tenemos dos formas de manejarlo:

Forma 1: .then() — la original
JS
// Estilo "cadena de promesas"
fetch('https://rickandmortyapi.com/api/character/1')
  .then(response => response.json())   // 1° parsea el JSON
  .then(data => console.log(data))       // 2° usa los datos
  .catch(error => console.error(error)); // si algo falla
Forma 2: async/await — la moderna (la que vamos a usar)
JS
// Parece código sincrónico pero es asincrónico
async function obtenerPersonaje() {
  const response = await fetch('https://rickandmortyapi.com/api/character/1');
  const data = await response.json();
  console.log(data);
}

// "await" detiene SOLO esa función, no todo el programa
// IMPORTANTE: solo podés usar await dentro de async function
💡 Regla de oro
Siempre que veas await, la función que lo contiene debe ser async. Si te olvidás del async, el código falla.
Bloque 1
fetch() —
tu puerta a las APIs

fetch() es la función nativa del navegador para hacer peticiones HTTP. Sin librerías externas, sin instalaciones. Solo JS.

La anatomía de fetch()

Estructura completa
JS
async function fetchPersonaje(id) {

  // 1️⃣  Hacemos la petición
  const response = await fetch(
    `https://rickandmortyapi.com/api/character/${id}`
  );

  // 2️⃣  Verificamos que salió bien
  if (!response.ok) {
    throw new Error(`HTTP error: ${response.status}`);
  }

  // 3️⃣  Convertimos la respuesta a objeto JS
  const data = await response.json();

  // 4️⃣  Usamos los datos
  return data;
}

Demo 2 — Cards de personajes

Ahora pedimos los primeros 8 personajes y los mostramos como tarjetas visuales. Aquí es donde map() y la manipulación del DOM se vuelven potentes.

Múltiples personajes + renderizado de cards
JS
async function mostrarPersonajes() {
  const url = 'https://rickandmortyapi.com/api/character';
  const response = await fetch(url);
  const data = await response.json();

  // data.results es el array de personajes
  const personajes = data.results.slice(0, 8);

  const container = document.getElementById('cards');

  container.innerHTML = personajes
    .map(p => `
      <div class="char-card">
        <img src="${p.image}" alt="${p.name}">
        <div class="char-info">
          <div class="char-name">${p.name}</div>
          <div class="char-meta">
            ${p.species} · ${p.gender}
          </div>
        </div>
      </div>
    `).join('');
}

Manejo de errores — siempre con try/catch

⚠ Error común
fetch() nunca falla aunque el servidor devuelva 404. El fetch fue exitoso — llegó una respuesta. Tenés que checar response.ok manualmente.
Patrón correcto de manejo de errores
JS
async function fetchSeguro(url) {
  try {
    const response = await fetch(url);

    if (!response.ok) {
      throw new Error(`Error ${response.status}: ${response.statusText}`);
    }

    return await response.json();

  } catch (error) {
    // Puede ser: red caída, URL inválida, error 404, etc.
    console.error('Falló la petición:', error.message);
    throw error; // re-lanzamos para que el caller lo maneje
  }
}
Bloque 2
Clases JS —
organizando el código

El código funcional funciona, pero cuando crece se vuelve un caos. Las clases nos permiten agrupar datos y comportamiento relacionados en un objeto reutilizable.

1
Clase más simple del mundo
Conceptos: constructor, propiedades, métodos
Clase básica
JS
class Personaje {
  // constructor: se ejecuta al hacer new Personaje()
  constructor(nombre, especie) {
    this.nombre = nombre;   // propiedad de instancia
    this.especie = especie;
  }

  // método: función que vive en la clase
  presentarse() {
    return `Soy ${this.nombre}, un ${this.especie}`;
  }
}

// Uso:
const rick = new Personaje('Rick', 'Humano');
console.log(rick.presentarse());
// → "Soy Rick, un Humano"
2
Encapsulando fetch en una clase
Ya empezamos a agrupar lógica de red
Clase con método async
JS
class APIClient {
  constructor(baseURL) {
    this.baseURL = baseURL;
  }

  // Los métodos de clase también pueden ser async
  async get(endpoint) {
    const response = await fetch(`${this.baseURL}${endpoint}`);
    if (!response.ok) throw new Error(response.status);
    return response.json();
  }
}

// Instanciamos con la base URL de la API
const api = new APIClient('https://rickandmortyapi.com/api');

// Usamos el método
const personaje = await api.get('/character/1');

Clase RickMortyAPI — completa y reutilizable
Métodos específicos por entidad, lógica de filtrado
RickMortyAPI.js
JS
class RickMortyAPI {
  // Propiedad estática: pertenece a la clase, no a la instancia
  static BASE_URL = 'https://rickandmortyapi.com/api';

  constructor() {
    this.cache = new Map(); // guardamos respuestas para no repetir fetch
  }

  async #fetchJSON(url) {         // # = método privado (ES2022)
    if (this.cache.has(url)) {
      return this.cache.get(url); // devuelve del cache si ya existe
    }
    const res = await fetch(url);
    if (!res.ok) throw new Error(`Error ${res.status}`);
    const data = await res.json();
    this.cache.set(url, data);
    return data;
  }

  // Métodos públicos
  async getCharacter(id) {
    return this.#fetchJSON(`${RickMortyAPI.BASE_URL}/character/${id}`);
  }

  async getCharacters(page = 1) {
    const data = await this.#fetchJSON(
      `${RickMortyAPI.BASE_URL}/character?page=${page}`
    );
    return data.results;
  }

  async searchByName(nombre) {
    const data = await this.#fetchJSON(
      `${RickMortyAPI.BASE_URL}/character?name=${nombre}`
    );
    return data.results;
  }
}

// Uso limpio:
const api = new RickMortyAPI();
const personajes = await api.getCharacters();
const rick = await api.getCharacter(1);

Bloque 3
Web Components —
el siguiente nivel

Un Web Component es un elemento HTML custom que definís vos. Una vez creado, podés usarlo como <mi-componente> en cualquier HTML. Es nativo del navegador — sin frameworks.

🏗️
Custom Elements
Registrás un elemento HTML nuevo. El navegador lo reconoce como una etiqueta válida.
🔒
Shadow DOM
Un DOM aislado. El CSS del componente no "contamina" la página y viceversa.
📋
<template>
HTML que se define pero no se renderiza hasta que lo clonás. El molde del componente.

Paso 1 — Custom Element mínimo

El Web Component más simple posible
JS
// 1. Creamos una clase que extiende HTMLElement
class MiSaludo extends HTMLElement {

  // connectedCallback: se ejecuta cuando el elemento
  // se agrega al DOM (como el DOMContentLoaded pero para el elemento)
  connectedCallback() {
    this.innerHTML = '<h2>¡Hola desde un Web Component!</h2>';
  }
}

// 2. Registramos el elemento (SIEMPRE con guión en el nombre)
customElements.define('mi-saludo', MiSaludo);

// 3. En el HTML lo usamos como una etiqueta normal:
// <mi-saludo></mi-saludo>

Paso 2 — Shadow DOM

Aislando el componente con Shadow DOM
JS
class MiBoton extends HTMLElement {
  constructor() {
    super(); // siempre primero

    // Creamos el shadow DOM
    const shadow = this.attachShadow({ mode: 'open' });

    // El CSS acá no afecta al resto de la página
    shadow.innerHTML = `
      <style>
        button {
          background: #00d4ff;
          border: none;
          padding: 8px 16px;
          border-radius: 4px;
          cursor: pointer;
          font-weight: bold;
        }
      </style>
      <button>Click!</button>
    `;
  }
}

customElements.define('mi-boton', MiBoton);

Paso 3 — Atributos reactivos

Leer atributos HTML desde el componente
JS
class MiTarjeta extends HTMLElement {

  // Declaramos qué atributos queremos observar
  static get observedAttributes() {
    return ['nombre', 'especie'];
  }

  connectedCallback() {
    this.render();
  }

  // Se llama cada vez que un atributo observado cambia
  attributeChangedCallback() {
    this.render();
  }

  render() {
    const nombre = this.getAttribute('nombre') || '???';
    const especie = this.getAttribute('especie') || '???';
    this.innerHTML = `<h3>${nombre} — ${especie}</h3>`;
  }
}

customElements.define('mi-tarjeta', MiTarjeta);

// En HTML:
// <mi-tarjeta nombre="Rick" especie="Humano"></mi-tarjeta>

todo junto

Acá está la versión completa: la clase RickMortyAPI + el Web Component <character-card> + un buscador funcional. Este es el código que van a llevar a casa.

Web Component completo — character-card.js
JS completo
// ═══ 1. La clase de API ═══
class RickMortyAPI {
  static BASE = 'https://rickandmortyapi.com/api';

  async getCharacters(page = 1) {
    const r = await fetch(`${RickMortyAPI.BASE}/character?page=${page}`);
    const d = await r.json();
    return d.results;
  }

  async search(nombre) {
    const r = await fetch(`${RickMortyAPI.BASE}/character?name=${nombre}`);
    if (!r.ok) return [];
    const d = await r.json();
    return d.results;
  }
}

// ═══ 2. El Web Component ═══
class CharacterCard extends HTMLElement {
  static get observedAttributes() {
    return ['char-id', 'name', 'image',
            'status', 'species', 'origin'];
  }

  constructor() {
    super();
    this.shadow = this.attachShadow({ mode: 'open' });
  }

  connectedCallback() { this.render(); }
  attributeChangedCallback() { this.render(); }

  render() {
    const g = (a) => this.getAttribute(a) || '—';
    const statusColor = {
      Alive: '#10b981', Dead: '#ef4444'
    }[g('status')] || '#64748b';

    this.shadow.innerHTML = `
      <style>
        :host { display: block; }
        .card {
          background: #1a2236;
          border: 1px solid #1e2d45;
          border-radius: 10px;
          overflow: hidden;
          transition: transform .2s, box-shadow .2s;
          font-family: 'Space Mono', monospace;
        }
        .card:hover {
          transform: translateY(-4px);
          box-shadow: 0 8px 30px rgba(0,212,255,.2);
        }
        img { width: 100%; display: block; }
        .info { padding: 12px; }
        .name {
          font-size: 13px;
          font-weight: 700;
          color: #e2e8f0;
          margin-bottom: 8px;
        }
        .row {
          display: flex;
          align-items: center;
          gap: 6px;
          font-size: 11px;
          color: #64748b;
          margin-bottom: 3px;
        }
        .dot {
          width: 7px; height: 7px;
          border-radius: 50%;
          background: ${statusColor};
          flex-shrink: 0;
        }
      </style>

      <div class="card">
        <img src="${g('image')}" alt="${g('name')}">
        <div class="info">
          <div class="name">${g('name')}</div>
          <div class="row">
            <span class="dot"></span>
            ${g('status')} · ${g('species')}
          </div>
          <div class="row">📍 ${g('origin')}</div>
        </div>
      </div>
    `;
  }
}

customElements.define('character-card', CharacterCard);

// ═══ 3. El controller ═══
const api = new RickMortyAPI();

async function renderizarPersonajes(personajes) {
  const grid = document.getElementById('grid');
  grid.innerHTML = personajes.map(p => `
    <character-card
      name="${p.name}"
      image="${p.image}"
      status="${p.status}"
      species="${p.species}"
      origin="${p.origin.name}"
    ></character-card>
  `).join('');
}

// Cargar al inicio
api.getCharacters().then(renderizarPersonajes);

// Buscador
document.getElementById('search').addEventListener('input', async (e) => {
  const q = e.target.value.trim();
  const resultados = q.length > 2
    ? await api.search(q)
    : await api.getCharacters();
  renderizarPersonajes(resultados);
});

Resumen
La evolución
en una vista
Etapa Código Qué ganamos Cuándo usarlo
1 Fetch suelto fetch(url).then(...) Funciona, rápido Script de una sola vez
2 async/await async function Más legible Siempre (sobre .then)

🎯 Para llevarse a casa
El camino siempre es el mismo: empezar simple, identificar qué se repite, y encapsular en una abstracción (función → clase → componente). La complejidad del código debería crecer solo cuando el problema lo justifica.
